Less-1

1.经典GET类型，使用 ?id=1 看有无数据回显，发现有回显，猜测是将id=1的参数传回后端数据库做搜索查询。

—+：可以理解为注释后面的语句

前端接收到?id的参数时，带回后端由数据库查询，如果我们输入的是1，则语句变为:

如果我们输入1'，则语句变为：
SELECT * FROM USERS WHERE ID = ‘1'‘ LIMIT 0,1;

前个引号相闭合，但是后面仍然多出一个引号，此时执行仍会报错，所以就需要—+ 将后面语句注释掉:

SELECT * FROM USERS WHERE ID = ‘1' --+‘ LIMIT 0,1;

2.使用 order by 猜解列数，当参数改为4时返回报错，得到该表有三个字段

3.使用联合查询 union select 查看有无回显点，发现2，3两处是回显点。

union select：联合查询。因为id一般从1开始，所以当id为-1时，查询失败，使用联合查询是因为当前面的语句查询失败时不影响后面语句执行，查询完毕后并返回执行结果。
